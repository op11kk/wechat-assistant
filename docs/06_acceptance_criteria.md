# 验收标准与里程碑（方案 B）

## 验收标准

### 对话主线

- 用户 **全程无需离开聊天即可完成**：参与引导 → 登记 → 获取编号与拍摄说明 → 在限制内 **直接发视频**。
- **不完整登记** 能追问缺失字段；**无效输入** 有统一 fallback 话术。

### 大视频补丁

- **菜单「大视频上传」** 可打开 H5，上传成功后 **3 秒内在后台可见** 一条 `video_submission`（H5 来源），且与用户编号一致。

### 媒体与数据

- 对话框产生的 `media_id` 均在过期前 **成功落盘** 对象存储，且库中有对应记录。
- **专属编号** 唯一；按编号可查提交列表与审核状态。

### 审核闭环

- 后台可将单条素材标为 **通过/驳回**，用户在微信侧收到对应通知（实现方式以所选消息能力为准）。
- 驳回原因可读，且可 **再次提交** 不产生编号冲突。

### 非功能

- 微信服务器 URL **验证 GET** 与消息 **POST 解密** 正常；服务 **HTTPS** 可用。
- 生产环境 **无** Token、EncodingAESKey、OSS 密钥入日志或仓库。

## 研发里程碑（建议）

| 阶段 | 内容 |
|------|------|
| M0 | 服务号认证、服务器配置、最小 echo 回调 |
| M1 | 用户登记 + 编号 + 媒体下载落 OSS + 审核最小闭环 |
| M2 | H5 大文件直传 + 与对话框同一套数据模型 |
| M3 | 定时提醒 / 模板消息、运营话术与关键词全量 |

## 异常与降级（示例）

| 场景 | 处理 |
|------|------|
| 超过被动回复时限 | 排队异步发送或使用合规的客服消息能力 |
| 下载 media 失败 | 重试 + 告知用户「请稍后重发或走大视频上传」 |
| H5 token 过期 | 重新从菜单进入，重新鉴权 |
| 用户要求删号 | 业务流程终止 + 按策略删库/脱敏对象 |
