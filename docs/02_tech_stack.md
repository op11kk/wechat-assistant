# 技术方案与选型（方案 B）

## 推荐架构（轻量）

| 层级 | 推荐选项 | 说明 |
|------|----------|------|
| 触点 | **微信认证服务号** | 消息接口 + 自定义菜单（少项） |
| 对话与规则 | 服务端回调 + **关键词 / 状态机**；可选接入 **AI Agent**（腾讯元器、自建大模型等） | 10 秒内响应目标与平台被动回复窗口需一并评估 |
| 大文件 | **菜单 → H5** | 浏览器直传 **Cloudflare R2**（PUT 预签名 URL，`POST /upload/presign`） |
| 媒体拉取 | 微信 **临时素材** API + **Cloudflare R2** | `WECHAT_APP_ID`/`WECHAT_APP_SECRET` 换 `access_token`，后台下载 `media_id` 后转存到 R2 |
| 业务数据 | 关系型 DB 或 **Supabase** 等 BaaS | 用户、编号、视频元数据、审核、收益台账 |
| 运维 | HTTPS 域名、日志、备份 | 微信服务器 URL 仅支持合法 TLS |

## 技术原则

1. **对话框优先**：接口设计以「用户发了一条什么消息 / 上传了什么」为第一事件源，而不是以页面 PV 为中心。
2. **编号与备注贯穿**：H5 与对话框共用同一 **专属编号 + 文案备注规范**，避免素材对不上人。
3. **敏感信息最小化**：仅收集收益与审核必需字段；隐私话术与用途在流程中明示。
4. **合规**：企业认证主体、用户同意、内容审核与微信平台运营规范需单独立账。

## 与小程序/APP 对比（为何当前选用 B）

- 用户 **无需新装 APP**；相较纯小程序方案，服务号 **对话 + 菜单跳 H5** 开发与迭代更轻，适合快速验证收集流程。
- 对话框对 **单文件大小** 有限制，故 **大视频必须走 H5 + R2 直传**，不在对话框硬扛。

## 云资源约定

- **对象存储**：仅使用 **Cloudflare R2**（通过 S3 兼容接口签发 PUT 预签名与写入对象）。
- **微信**：公众平台与服务号主体可独立；转存链路为 **微信临时素材 → 本服务拉流 → Cloudflare R2**。
