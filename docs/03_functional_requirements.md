# 功能需求（方案 B：服务号 · 对话主线）

## 1. 服务号与入口

- 完成 **微信认证（服务号）**，启用 **服务器配置**（URL、Token、EncodingAESKey）。
- **自定义菜单**：V1 建议 ≤3 个一级项，至少包含 **「大视频上传」**（打开 H5）；其余不放复杂功能树。
- **可选**：关注后 **欢迎语**（与 AI/规则引擎首条话术一致，避免重复矛盾）。

## 2. 对话内流程（主干）

以下环节主要通过 **用户发文字 / 视频 / 图片（若需要）** 与 **公众号被动回复 / 客服能力** 完成，顺序与话术以运营定稿为准。

| 环节 | 触发 | 系统行为 |
|------|------|----------|
| 冷启动 | 关注 / 首条指令 | 说明背景、合规用途、收益原则、隐私；引导回复关键词（如「我要参与」） |
| 登记 | 关键词 + 结构化对话 | 收集必填字段（如姓名、电话）；校验不完整则追问 |
| 配号与需求 | 登记成功 | 生成 **专属编号**；下发拍摄与格式要求（时长、分辨率、主题禁忌等） |
| 答疑 | 关键词 / 意图 | 覆盖收益、隐私、审核周期、上传失败、修改信息等；无法命中则引导人工时段 |
| 收视频 | 用户发视频消息或 H5 提交完成回调 | 绑定 `openid` + 编号 + 备注；生成「已收悉、审核中」话术 |
| 审核反馈 | 人工在后台标记结果 | 自动推送通过/不通过及原因；通过可关联结算说明与「是否继续参与」 |
| 周期触达 | 定时任务 | 对「愿意继续参与」的用户推送新需求或截止提醒（需符合微信模板/订阅规则） |

## 3. 大视频上传（菜单 / H5 仅作补丁）

- H5 需支持：**登录态或一次性带参令牌**（与 openid、编号绑定）、分片或直传、进度与成功页。
- 成功后 **回调业务后端** 写入一条与对话框上传 **同结构** 的「待审核」记录。

## 4. 后台与管理

- **审核台**：列表、预览、通过/驳回、驳回原因模板。
- **用户与编号**：按 openid、编号查询、限次修改信息等业务规则可配置。
- **只读统计**：提交量、通过率、响应时效（可选 V2）。

## 5. 非功能

- 自动回复 **延迟目标**（如 ≤10s）与微信 **5s 被动回复** 等限制冲突时，需 **异步队列 + 客服消息** 等备选方案（实施时选型）。
- 全链路 **HTTPS**；密钥与 Token 不入库明文日志。
