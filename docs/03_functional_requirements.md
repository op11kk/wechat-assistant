# 功能需求

## 模式设计

### 模式定义
| 模式 | 适用设备 | 输出内容 |
|------|----------|----------|
| Full Capture | 支持 LiDAR + scene depth | RGB + Depth + IMU + Pose |
| Lite Capture | 基础相机 + IMU + tracking | RGB + IMU + Pose |

### 模式选择逻辑
1. 先检测设备能力
2. 再展示可用模式
3. 由用户主动选择（不自动降级）

### 交互要求
- 支持 Full 的设备：用户可在 Lite / Full 间切换
- 不支持 Full 的设备：Full 置灰锁定，仅可选 Lite
- 用户选 Full 但初始化失败：明确提示，允许改选 Lite
- 用户选 Lite：始终按 Lite 初始化，不强制升级

## 核心体验

### 录制前
- 权限检查（相机、运动传感器、存储）
- 能力检测（tracking、depth、性能、热状态）
- 模式选择（显示可用性与说明）
- 参数确认（时长、存储、模式说明）

### 录制中
- 实时预览
- 显示当前模式、时长、RGB fps
- 显示 tracking 状态、depth 状态（仅 Full）
- 热状态与存储提示
- 支持开始、暂停、恢复、停止

### 录制后
- 自动生成本地 session
- 会话列表查看、预览、重命名、删除、上传
- 上传失败后支持重试

## 本地会话管理
| 功能 | 要求 |
|------|------|
| 会话列表 | 展示缩略图、模式、时长、大小、上传状态 |
| 详情页 | 展示帧数、样本数、模式、设备、异常摘要 |
| 重命名/删除 | 支持 |
| 上传/重试 | 支持手动触发及失败重试 |
